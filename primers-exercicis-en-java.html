Primers exercicis en Java

Entrada ubicada a la categoria poo amb el tag: java

Introducció

Aquest document ofereix l'estudiant la possibilitat d'accedir a la sintaxi bàsica del Java, a partir d'una sèrie d'exercicis i pistes.

Pel seu seguiment es pressuposa coneixements de programació estructurada i modular, així com de la sintaxi bàsica del llenguatge C.

Exercicis

Considera les pistes i realitza els exercicis següents.

No cal preocupar-se, de moment, del control d'errors per entrades de tipus erroni. Suposarem que els nostres usuaris sempre introduiran els valors que els demanem del tipus correcte.

Pista 1. El nom del fitxer

El fitxer que conté un programa en Java, ha d'anomenar-se exactament igual que la classe. Així, el programa HolaMon.java tindrà la següent estructura:

public class HolaMon { ... }

Pista 2. El bloc principal

la rutina main() del llenguatge C, té la seva equivalència amb

public class NomClasse {
    public static void main(String[] args) {...}
}

Pista 3. Escriure per sortida estàndard

A l'hora d'escriure per sortida estàndard, les següents maneres són equivalents:

System.out.println("missatge seguit de salt de línia");
System.out.print("missatge seguit de salt de línia\n");

/*
    Descr: programa que demana un número enter i mostra el seu quadrat.
 */
import java.io.*;

public class Quadrat {
  public static void main(String[] args) throws IOException {
    System.out.print("Número: ");
    int num = Integer.parseInt(new BufferedReader(new InputStreamReader(System.in)).readLine());
    System.out.println("El valor de " + num + " al quadrat és " + (num * num));
  }
}

Atenció: aquest programa no controla el cas que l'usuari escrigui quelcom que no sigui un número enter. Prova què passa si ho fa.

Si cada cop que volem llegir quelcom de l'entrada estàndard hem de fer tot això, a banda de ser quelcom poc eficient tant respecte a l'hora de codificar com d'executar, ens trobarem amb problemes a l'hora de redireccionar l'entrada.

import java.io.*;
public class IOUtils {
    private static BufferedReader br = new BufferedReader(
            new InputStreamReader(System.in));
    // retorna un enter llegit de entrada estàndard. -1 si error
    public static int llegeixInt() {
        int resposta = -1;
        try {
            resposta = Integer.parseInt(llegeixStr());
        } catch (Exception e) {}
        return resposta;
    }
    // retorna un String llegit de entrada estàndard. "" si error.
    public static String llegeixStr() {
        String resposta = "";
        try {
            resposta = br.readLine();
        } catch (Exception e) {}
        return resposta;
    }
}

Pista 5. Sintaxi similar al C

La sintaxi del Java està molt inspirada en el llenguatge C. Per aquesta raó, si no saps com s'escriu alguna cosa en java, prova amb la sintaxi de C. Si tot i així no te'n surts, és clar, et caldrà una mica més de cerca.

Exercici 1. Presó

Desenvolupa un programa que demani a l'usuari els anys que té amb el missatge Quants anys té?. En cas que sigui major d'edat, el programa li avisarà dient: Atenció: Vosté pot anar a la presó!. En qualsevol cas, el programa finalitzarà dient: Aneu amb compte.

Anomena el fitxer amb el codi font Preso.java.

Nota: en aquest cas acceptarem qualsevol valor enter com a edat vàlida.

Exercici 2. Mesos

Desenvolupeu un programa que demani un mes i un any, i escrigui el mes anterior i el mes següent.

Els missatges de petició de l'entrada seran, respectivament: Mes? i Any?.

Per exemple, si l’usuari introdueix mes 9 i any 2009, el resultat serà:

ant: 8/2009
seg: 10/2009

Nota: en aquest cas, suposarem que sempre ens introduiran mesos i anys correctes.

Anomena el fitxer amb el codi font Mesos.java.

Exercici 3. Ordena 3

Desenvolupeu un programa que demani tres números i escrigui els tres números ordenats creixentment (de menor a major).

L'entrada es requerirà amb els missatges Primer?, Segon? i Tercer?.

Per exemple, si els números introduïts són 3, 4 i 2, el resultat serà:

2 3 4

Exercici 4. Ascensor

Considerem un ascensor d’un edifici amb planta baixa i dos pisos que tingui els següents botons: ‘pujar 1’, ‘pujar 2’, ‘baixar 1’ i ‘baixar 2’. L’ascensor es comporta, a partir dels botons esmentats, segons el següent diagrama de transició d’estats:

Es demana desenvolupar un programa que simuli el funcionament d’aquest ascensor. El programa demanarà el pis en que es troba i el botó que es prem, i mostrarà el nou pis. En cas que el botó seleccionat a una posició determinada de l’ascensor no correspongui amb algun dels representats amb el diagrama d’estats, l’ascensor mostra un missatge d'error i es queda al lloc en què estava.

Els missatges de petició de l'entrada són, respectivament:

Planta? (PB: planta baixa, PP: primer pis, SP: segon pis
Botó? (P1: puja 1, P2: puja 2, B1: baixa 1, B2: baixa 2

Mentre que la sortida s'expressarà simplement indicant la nova planta.

Per exemple, si l'entrada és: PB i P1, la sortida serà:

PP

En cas d'error, caldrà mostrar els següents missatges:

Planta incorrecta
Botó incorrecte

I cal tenir present que si la planta és incorrecta, ja no se li demanarà el botó a l'usuari.

Anomena el fitxer amb el codi font Ascensor.java.

Exercici 5. Històric d'edats

Desenvolupa un programa que demani el nom de l’usuari, l’edat i l’any actual. El programa escriurà l’edat que tenia l’usuari a cada any des del seu naixement i s’acomiadarà. Per exemple, si l’entrada és

Anna
5
2012

El programa escriurà

El 2007 va nàixer
El 2008 tenia 1 any
El 2009 tenia 2 anys
El 2010 tenia 3 anys
El 2011 tenia 4 anys
Adéu Anna

L'aplicació no li dirà l'edat que té l'usuàri en l'any actual, donat que aquesta dada forma part de l'entrada. Per exemple, si l'usuària ha nascut aquest any, simplement s'acomiadarà.

Les comprovacions necessàries són: que el nom no estigui buit, que l'edat i l'any actual siguin enters positius, i que l'any actual sigui un valor més alt que el de l'edat.

Es demanaran les dades en l'ordre indicat a l'exemple i, en quan es detecti un error, es deixarà de demanar més dades.

Els missatges de petició de dades seran:

Nom?
Edat?
Any actual?

Els missatges d'error seran:

Nom incorrecte
Edat no pot ser negativa
Any actual no pot ser negatiu
Edat o any actual incorrectes

Anomena el fitxer amb el codi font Edats.java.

/*
    Descr: programa que va demanant paraules fins que l'indiquen "prou" i les
    mostra en majúscules
    Nota: requereix IOUtils al package per defecte.
 */
import java.io.*;

public class PosaMajuscules {
    public static void main(String[] args) throws IOException {
        final String PARAULA_FIN = "prou"; // constant amb la paraula amb que acaba
        boolean continua = true;
        while (continua) {
            System.out.print("Mot a posar en majúscules ("+PARAULA_FIN+" acaba): ");
            String mot = IOUtils.llegeixStr();
            if (mot.equals(PARAULA_FIN)) {
                continua = false;
            } else {
                System.out.println("El mot " + mot + " en majúscules és " + mot.toUpperCase());
            }
        }
    }
}

Exercici 6. L'ascensor continu

Ara, el nostre ascensor començarà a la planta baixa i anirà demanant un botó per canviar de pis. Es comportarà pràcticament igual que la versió anterior. Ara, és clar, ja no li cal demanar a quin pis es troba.

Un cop canviat de pis (o indicat l'error) demanarà que premi un nou botó. I així es quedarà fins que l'usuari indiqui el botó de sortida.

Aquesta versió incorpora un missatge molt compacte amb el següent aspecte:

[xx] P1: puja 1, P2: puja 2, B1: baixa 1, B2: baixa 2, EX: surt

On xx pot ser PB, PP o SP depenent de la planta on es trobi.

Quan l'usuari demani finalitzar (amb "EX"), es mostrarà el missatge de comiat "Adéu" i finalitzarà l'execució.

Anomena el fitxer amb el codi font AscensorContinu.java.

/*
    Descr.: dibuixa un triangle n cops
 */
public class Triangle {
    private static final String PARAULA = "Triangle";
    public static void main(String[] args) {
        if (args.length < 1) {    // nombre d'arguments passats
            System.out.println("Per favor, indiqueu el " +
                "nombre de triangles a dibuixar. Ex.");
            System.out.println("\tjava Triangle 3");
        } else {
            int num = Integer.parseInt(args[0]); // primer argument
            for (int i=0; i<num; i++) {
                dibuixa(i);                      // crida a la rutina dibuixa()
            }
        }
    }
    private static void dibuixa(int num) {       // declara la rutina dibuixa()
        /* dibuixa el triangle número num */
        System.out.println("Aquest és el triangle número " + (num + 1));
        for (int l=0; l<PARAULA.length(); l++) {
            for (int c=0; c<=l; c++) {
                System.out.print(PARAULA.charAt(c)); // escriu el caràcter a la posició c
            }
            System.out.println();
        }
    }
}
