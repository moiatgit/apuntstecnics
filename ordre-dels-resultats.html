Ordre dels resultats

Entrada ubicada a la categoria mongodb amb el tag: mongoDB

Introducció

Fins ara hem vist com consultar dades, amb la possibilitat de filtrar-les i només rebre aquelles que volem. No hem parlat gens de en quin ordre les volem. Aquesta és la finalitat del mètode sort i presentar-te'l ho és d'aquest article.

Dades ordenades

Considera els resultats de la següent consulta:

> db.persones.find({"professió":"desenvolupador/a"}, {"nom":true, "sou":true, "_id":false})
{ "nom" : "Irene Ruíz Cloquell", "sou" : 2500 }
{ "nom" : "Iker Navarro López", "sou" : 2800 }
{ "nom" : "Pol Fernández García", "sou" : 1800 }

Tot i que a tu et podrien sortir amb un ordre diferent, fixa't com el resultat són els documents requerits però amb un ordre aparentment independent de les claus sol·licitades.

Suposa que ens interessés disposar de les dades en ordre ascendent de sou. Fixa't en la següent consulta:

> db.persones.find({"professió":"desenvolupador/a"}, {"nom":true, "sou":true, "_id":false}).sort({sou:1})
{ "nom" : "Pol Fernández García", "sou" : 1800 }
{ "nom" : "Irene Ruíz Cloquell", "sou" : 2500 }
{ "nom" : "Iker Navarro López", "sou" : 2800 }

El mètode sort rep un document on la clau correspon a claus dels documents de la col·lecció consultada, i el valor és:

  • 1: per indicar ordre ascendent
  • -1: per indicar ordre descendent

Així, simplement canviant el signe, tenim l'ordre canviat:

> db.persones.find({"professió":"desenvolupador/a"}, {"nom":true, "sou":true, "_id":false}).sort({sou:-1})
{ "nom" : "Iker Navarro López",     "sou" : 2800 }
{ "nom" : "Irene Ruíz Cloquell",    "sou" : 2500 }
{ "nom" : "Pol Fernández García",   "sou" : 1800 }

Podríem estar interessats en controlar l'ordre d'altres claus. Al següent exemple ampliarem la consulta a tots els documents i ordenarem primer per sou i després per edat:

Exercici 1. Primer els grans

Composa una consulta que retorni les persones per ordre d'edat, de manera que la primera sigui la més gran i la darrera la més jove.

Fes que només apareguin el nom i l'edat.

Exercici 2. Primer els joves

Composa una consulta que retorni les persones per ordre d'edat, de manera que la primera sigui la més jove i la darrera la més gran.

Si més d'una persona té la mateixa edat, fes que primer surti la que tingui el sou més alt.

Fes que només apareguin el nom, l'edat i el sou.

Aprofitar l'ordre per filtar

Tot conjugant el sort amb skip i limit ens permet aconseguir consultes com ara, "les tres persones amb millor sou" o "les dues persones més joves després de la més jove".

De seguida et mostraré com fer-les, però et recomano que les intentis fer pel teu compte abans de continuar llegint. Et permetrà comprovar si estàs agafant la filosofia de MongoDB.

Les tres persones amb millor sou

Per a aconseguir aquesta consulta simplement cal ordenar per sou (primer el més alt) i quedar-te amb només tres.

> db.persones.find().sort({"sou":-1}).limit(3)

Exercici 3. Les 2 persones que cobren menys

Composa una consulta que retorni les dues persones amb el sou més baix.

Exercici 4. Les 4 persones menys joves

Composa una consulta que retorni només les quatre persones amb més edat.

Les dues persones més joves després de la més jove

Aquesta consulta ens requereix fer servir també el mètode skip amb el que ens saltarem la més persona més jove.

> db.persones.find().sort({"edat":1}).skip(1).limit(2)
{ "_id" : ObjectId("53038dcd06facaddfaf743a3"), "nom" : "Carla Fernández Balló", "edat" : 20, "professió" : "pintor/a", "sou" : 2400 }
{ "_id" : ObjectId("53038dcd06facaddfaf7439e"), "nom" : "Iker Fernández Fijo", "edat" : 21, "professió" : "fotògraf/a", "sou" : 2200 }

Exercici 7. No volem les més joves
