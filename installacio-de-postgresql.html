Instal·lació de PostGreSQL

Entrada ubicada a la categoria bd amb el tag: PostGreSQL

Introducció

Aquesta guia descriu, pas a pas, una instal·lació bàsica del Sistema Gestor de Bases de Dades (SGBD) PostGreSQL sobre una màquina virtual amb un servidor Ubuntu.

Es recomana la instal·lació del servidor de PostGreSQL a una màquina independent de la màquina de treball però connectada a aquesta. D'aquesta manera les proves que es realitzin s'acostaran més a una manera de treballar més real. Per simplicitat/economia, es proposa que el servidor sigui una màquina virtual.

En aquest document suposarà que disposes d'una màquina Ubuntu que hostatjarà la màquina virtual d'un servidor Ubuntu gestionada per l'aplicació de virtualització *VirtualBox*.

Pel cas que no coneguis l'aplicació VirtualBox o/i com crear una màquina virtual, aquest document et donarà unes breus indicacions per a aconseguir configurar quelcom usable.

En el cas que prefereixis fer servir algun altre sistema operatiu o bé una aplicació de virtualització diferent, no hauries de tenir gaires problemes per a seguir el principal objectiu d'aquest document: la instal·lació d'un servidor PostGreSQL.

Creació de la màquina virtual

A continuació es presenta una breu guia per a crear una màquina virtual amb el servidor des d'un equip Ubuntu.

Instal·lació del VirtualBox

A la pàgina Download VirtualBox for Linux Hosts hi trobaràs diferents maneres d'instal·lar el VirtualBox a una màquina GNU/Linux.

Instal·la l'aplicació a la màquina on realitzaràs les pràctiques d'aquest document (a partir d'ara amfitrió). Si l'amfitrió és un Ubuntu 12.04, les instruccions següents t'ho permetran fàcilment:

  1. Afegeix el repositori

    Afegeix la següent línia al final del fitxer /etc/apt/sources.list

    deb http://download.virtualbox.org/virtualbox/debian precise contrib

  2. Registra la clau pública d'Oracle:

    $ wget -q http://download.virtualbox.org/virtualbox/debian/oracle_vbox.asc -O- | sudo apt-key add -

  3. Actualitza la base de dades de apt i instal·la l'aplicació:

    $ sudo apt-get update
$ sudo apt-get install virtualbox-4.3

  4. Arrenca l'aplicació:

    $ VirtualBox &

Creació del servidor

A continuació trobaràs unes instruccions per a crear una màquina virtual bàsica i instal·lar-li un servidor d'Ubuntu. Si saps el que estàs fent, hauries de poder realitzar-ho d'altres maneres sense que això afecti l'objectiu d'aquesta lliçó.

Ara crearem ara una màquina virtual amb un Ubuntu Server. Segueix les passes:

  1. Descarrega la imatge amb l'instal·lador del servidor d'Ubuntu.

    Descarrega la imatge d'una versió del servidor d'Ubuntu a la pàgina Download Ubuntu Server

  2. Crea una nova màquina virtual

    Arrenca VirtualBox i selecciona Màquina->Nova (o ctrl-N)

    Indica un nom per la màquina (ex. Ubuntu PostGreSQL) de tipus Linux i versió Ubuntu. Reserva la memòria recomanada i crea un nou disc virtual de tipus VDI, ubicat de forma dinàmica i amb la ubicació i mida que se't proposin. És a dir, accepta tots els valors que et proposen per defecte.

  3. Enllaça la imatge del servidor amb la nova màquina virtual

    Per fer-ho, segueix Màquina->Paràmetres->Emmagatzematge, clica sobre la icona del CD a l'àrbre d'emmagatzematge i tot seguit a la mateixa icona però a la secció d'atributs. Al desplegable, selecciona la opció de Selecciona una imatge de CD/DVD virtual i navega fins a seleccionar el fitxer amb la imatge que t'has descarregat.

  4. Arrenca la màquina i instal·la el sistema operatiu.

    Segueix Màquina->Inicia i segueix les instruccions per instal·lar el servidor d'Ubuntu.

    En cas que no tinguis familiaritat amb la instal·lació no gràfica d'un sistema operatiu com l'Ubuntu Server, simplement escull idioma [1] i selecciona Install Ububtu Server. Selecciona els valors que consideris adequats (el del teclat potser voldràs que te l'intenti detectar l'instal·lador)

    A continuació creat un usuari sudoer i posa-li una contrasenya que puguis recordar. Aquí suposarem usuarivm. També et caldrà definir un nom de la màquina. Aquí suposarem vm.

    Continua amb la resta d'opcions fins la creació de les particions. Normalment el valor proposat hauria de ser adequat. Amb la secció de creació de particions escull la primera opció Guided - use entire disk (sense LVM) i accepta la taula de particions que t'ofereixi.

    Ara espera a que s'instal·li el sistema operatiu.

    Hi haurà algunes preguntes que requeriran de la teva interacció. La major part de les vegades podràs seleccionar l'opció per defecte fins que arribis a la instal·lació de paquets de la secció Software selection. Si et vols evitar la instal·lació posterior del servidor de PostGreSQL i del de SSH (per exemple per fer login remotament), pots aprofitar i marcar OpenSSH Server i PostgreSQL database.

    No és imprescindible instal·lar el servidor de ssh, però pot resultar-te molt més còmode establir sessió des d'una consola a l'amfitrió que teclejar directament a la màquina virtual. Com tot, va a gustos. Si no vas marcar el paquet OpenSSH, però t'interessa més endavant, el podràs instal·lar amb un:

    usuarivm@vm:~$ sudo apt-get install openssh-server

  5. Un cop instal·lat el sistema operatiu, torna a arrencar la màquina. Per exemple, amb:

    usuarivm@vm:~$ sudo shutdown -r now

    Entra amb el teu usuari sudoer (si has seguit el nom d'aquesta guia, serà usuarivm) i actualitza (si tens temps) el programari amb:

    usuarivm@vm:~$ sudo apt-get uptdate
usuarivm@vm:~$ sudo apt-get upgrade

  6. Canvia la connexió de la xarxa

    El mode de connexió NAT ens permet accedir a l'exterior (internet) des de la màquina virtual. Ens limita, però, l'accés des de la màquina host al servidor: precisament el que més ens interessa.

    Per aquesta raó, canviarem el tipus de connexió a un altre que ens ajudi al nostre objectiu. Escollirem el mode Adaptador de només amfitrió de manera que l'accés a la màquina virtual quedi restringit a la nostra màquina i altres possibles màquines virtuals que s'executin sobre aquesta.

    Per fer-ho, segueix les següents passes:

    1. finalitza l'execució de la màquina virtual

    2. des de la finestra principal de VirtualBox, selecciona Fitxer->Preferències->Xarxa->Xarxes de només amfitrió i fes clic al botó lateral (dreta) que apareix amb un '+' (amb llegenda Âfegeix xarxa de només anfitrió). Això t'incorporarà vboxnet0, una xarxa que crea VirtualBox.

      És possible que no disposis de vboxnet0 inicialment. Per crear-la, hauràs d'anar a la finestra principal de VirtualBox i seguir el camí Fitxer->Preferències->Xarxa. Ara, a la dreta de la finestra trobaràs una icona per afegir una altra interfície de xarxa. Prem-la i et crearà vboxnet0. En pots crear-ne més però probablement no et caldrà per a aquesta pràctica.

    3. des de la finestra principal de VirtualBox, selecciona Màquina->Paràmetres->Xarxa i a Connectat a: canvia el valor de NAT per Adaptador de només amfitrió [2].

    Cal tenir en compte que en arrencar de nou la màquina virtual haurem perdut l'accés cap a l'exterior. Si et cal per algun motiu, pots provar, per exemple, d'assignar a la màquina virtual una interfície de xarxa addicional amb NAT [3]. Pels objectius d'aquest document, no serà necessari.

    Ara, pots consultar la informació de xarxa a la màquina virtual:

    usuarivm@vm:~$ ifconfig eth0

          TX packets:46 errors:0 dropped:0 overruns:0 carrier:0

    També des de la màquina amfitrió, on comprovarem que VirtualBox ens ha creat una nova interfície anomenada vboxnet0:

    $ ifconfig vboxnet0

          inet addr:192.168.56.1  Bcast:192.168.56.255  Mask:255.255.255.0

    Podem ara, per exemple, connectar-nos a la màquina virtual amb una sessió ssh des d'una consola de l'amfitrió:

    $ ssh usuarivm@192.168.56.101

Instal·lació de PostGreSQL

Aquesta secció només considera-la en el cas que no hagis ja instal·lat el PostGreSQL com una de les col·leccions d'aplicacions durant la instal·lació del SO.

La instal·lació del servidor de PostGreSQL en la configuració proposada no podria ser gaire més senzilla. Assegura't però que ho fas a la màquina virtual i no a la amfitriona! :

usuarivm@vm:~$ sudo apt-get install postgresql

Segurament no t'instal·larà la darrera versió de PostGreSQL, però al menys serà una estable i suficient pels objectius d'aquesta lliçó.

Configuració de PostGreSQL

En instal·lar PostGreSQL, ens ha creat usuari postgres i la base de dades template1.

En cas que sigui necessari, canviarem la contrasenya de postgres amb:

usuarivm@vm:~$ sudo passwd postgres

Per defecte, PostGreSQL configura l'accés local perquè sigui realitzat des d'un usuari del sistema operatiu. Tot i que es pot canviar, acceptarem aquest comportament per defecte.

Així, crearem un nou usuari al sistema. Ho farem sense donar-li possibilitats de logar-se:

usuarivm@vm:~$ sudo useradd -s /sbin/nologin usuaribd

Creem ara un usuari regular i una nova base de dades. Farem:

usuarivm@vm:~$ su - postgres
postgres@vm:~$ psql
$ psql
psql (9.3.6)
Type "help" for help.
postgres=# CREATE USER usuaribd WITH PASSWORD 'toPSikr3t';
CREATE ROLE
postgres=# CREATE DATABASE testbd;
CREATE DATABASE
postgres=# GRANT ALL PRIVILEGES ON DATABASE testbd to usuaribd;
GRANT
postgres=#\q
postgres@vm:~$ exit
logout

Nota: és possible que el primer cop que sortim de psql ens generi un missatge com ara:

could not save history to file
"/var/lib/postgresql/.psql_history": No such file or directory

Si és així, el podem ignorar amb tranquil·litat.

Evidentment pots escollir el nom de la base de dades, el d'usuari i la clau que t'agradin. Recorda que si la configuració de xarxa de la teva màquina virtual és la que se't proposa en aquest document, només es podrà accedir a aquesta des de la teva màquina amfitrió.

Accedim a la nova base de dades amb el nou usuari fent servir:

usuarivm@vm:~$ psql -U usuaribd -d testbd -h localhost
Password for user usuaribd:
psql (9.3.6)
SSL connection (cipher: DHE-RSA-AES256-SHA, bits: 256)
Type "help" for help.

testbd=>\q

Per a accedir des d'una màquina fora de la que conté el servidor, per exemple del nostre amfitrió [4], ens caldrà fer algunes modificacions a la configuració per defecte de PostGreSQL.

El canvi requereix la modificació de dos fitxers de configuració que es trobaran a /etc/postgresql/«X.Y» on «X.Y» correspon a la versió que hagis instal·lat de PostGreSQL. En el meu cas la 9.3.

El primer fitxer a modificar és /etc/postgresql/9.3/main/pg_hba.conf. Cal localitzar les següents línies

# IPv4 local connections:
host    all             all             127.0.0.1/32            md5

Tot just a sota, hi afegirem la següent línia [5]

host    all             all             192.168.56.0/24        md5

A continuació, modificarem el fitxer /etc/postgresql/9.3/main/postgresql.conf. Hi localitzarem la secció Connection settings i descomentarem la línia que conté

# listen_addresses = 'localhost'          # what IP address(es) to listen on;

de manera que aparegui com:

listen_addresses = '*'          # what IP address(es) to listen on;

Ara reiniciem el servidor de PostGreSQL amb:

usuarivm@vm:~$ sudo service postgresql restart

Finalment provarem la connexió remota. Per simplicitat, farem servir la consola de psql des de l'amfitrió.

A la màquina amfitrió ens caldrà probablement instal·lar el client de PostGreSQL. Ho podem fer simplement amb:

$ sudo apt-get install postgresql-client

Ara ja podem fer una prova des de l'amfitrió:

$ psql -U usuaribd -d testbd -h 192.168.56.101
Password for user usuaribd:
psql (9.3.6)
SSL connection (cipher: DHE-RSA-AES256-GCM-SHA384, bits: 256)
Type "help" for help.

testbd=> \q

Si tot ha anat bé, ja disposem d'un servidor PostGreSQL en funcionament amb un usuari i una base de dades amb els que podrem estudiar el funcionament d'aquest SGBD.

Notes

[1]En aquest cas, l'idioma és només per l'instal·lador. De seguida et demanarà l'idioma del SO. Et recomano que mantinguis l'anglés de manera que no hagis de barallar-te amb la instal·lació de diferents paquets de traducció.
[2]Pot ser temptador fer servir l'opció d'Adaptador pont. Si et trobes però a una xarxa amb molts equips, tingues en compte que aquesta opció consumirà una IP del servidor DHCP. A més, les comunicacions entre la màquina virtual i l'amfitrió seran més lentes donat que cal que surtin físicament.
[3]

En cas que vulguis configurar més d'una interfície de xarxa, és possible que no sigui suficient amb configurar els dos adaptadors al VirtualBox. Hauràs de configurar la màquina virtual de manera que reconegui la nova interfície. Per fer-ho, considera afegir les següents línies al final de /etc/network/interfaces

auto eth1
iface eth1 inet dhcp
[4]Per favor, no et confonguis amb el fet d'estar-te connectant des d'una sessió ssh. Tot i que la connexió ssh l'estàs realitzant des de la màquina anfitriona, la connexió al servidor de PostGreSQL l'estàs realitzant des de la màquina virtual.
[5]Per editar, el servidor us ofereix dos editors: el vi i el nano. Si no saps manegar-te amb el primer, els ajuts del segon t'haurien de ser suficients.
